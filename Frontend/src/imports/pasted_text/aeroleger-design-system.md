Design a premium, modern UI system and full product experience for a web app called AeroLedger.

Product context:
AeroLedger is a crypto-native aviation commerce platform for two luxury use cases:
1. booking private or commercial flights
2. browsing and purchasing high-end aircraft

The platform accepts cryptocurrency payments and supports BTC, ETH, USDT, USDC, SOL, and BNB. It includes booking flows, aircraft marketplace pages, checkout, payment confirmation with transaction hash, user dashboards, admin views, and transaction tracking.

Primary goal:
Create a polished, high-end, investor-ready product design that feels exclusive, trustworthy, futuristic, and easy to use. The interface should look like a luxury aviation brand blended with a premium fintech/crypto product. Avoid generic SaaS styling. The experience should feel cinematic, elegant, and expensive.

Visual direction:
- A luxury aviation aesthetic with strong editorial composition
- Clean futuristic fintech details without looking cold or overly technical
- High contrast, rich depth, refined spacing, and clear hierarchy
- Premium typography and dramatic hero sections
- Smooth modern layout with subtle glass, metal, carbon-fiber, runway-light, or cockpit-inspired influences
- Use a dark or deep-neutral base with refined accent colors inspired by aviation and wealth
- Avoid childish crypto visuals, meme aesthetics, neon overload, or generic startup gradients
- Make it feel like a brand for ultra-high-net-worth clients

Brand personality:
- Exclusive
- Trustworthy
- Sophisticated
- Fast
- Global
- Secure
- High-touch concierge-level service

Target users:
- Wealthy travelers booking private flights
- Buyers browsing luxury aircraft
- Crypto-native clients who want premium experiences
- Admin/operators managing bookings, payments, and transactions

Deliverables:
Create a complete design system and key product screens for desktop first, then responsive mobile versions.

Include these design foundations:
- Color palette with primary, secondary, accent, surface, border, success, warning, error colors
- Typography system with display, heading, subheading, body, caption, and numeric styles
- Button system: primary, secondary, ghost, destructive
- Form controls: inputs, dropdowns, date pickers, segmented controls, checkboxes, radio buttons
- Cards, tables, tabs, modals, drawers, alerts, badges, toasts
- Navigation patterns for public pages and authenticated dashboards
- Spacing, grid, corner radius, shadows, and elevation rules
- Iconography direction
- Light motion guidance for hover, transitions, reveals, loading states
- Empty states, loading states, and error states

Design these screens and flows:

1. Landing page
- Luxury hero section with strong brand statement
- Navigation with sections for Flights, Aircraft Marketplace, Portfolio, Dashboard, Sign In
- Clear CTAs for “Book a Flight” and “Browse Aircraft”
- Section explaining crypto payment convenience and supported assets
- Showcase featured aircraft and premium flight options
- Trust section for security, verification, and global support
- Elegant footer

2. Flights search page
- Search/filter panel for origin, destination, passenger count, cabin class, trip type, date
- Results cards with pricing, aircraft/airline details, duration, highlights
- Premium but practical layout
- Filter and sort interactions
- Strong mobile treatment

3. Aircraft marketplace listing
- Luxury catalog view for aircraft sales
- Rich cards with aircraft imagery, pricing, specs, year, range, seating, manufacturer
- Filters for manufacturer, range, price, condition, category
- Premium browsing experience with strong visual hierarchy

4. Aircraft detail page
- Large immersive hero gallery
- Aircraft overview, specs, performance, highlights, and pricing
- Inquiry or purchase CTA
- Clean specification tables and sections
- Related aircraft recommendations

5. Booking/checkout flow
- Multi-step layout that feels premium and secure
- Booking summary
- Passenger/client information
- Pricing breakdown
- Crypto payment selection
- Wallet/payment instructions
- Transaction hash submission
- Confirmation state

6. Crypto payment UI
- Clearly show supported currencies: BTC, ETH, USDT, USDC, SOL, BNB
- Payment amount, destination wallet, QR area, network information
- Status progression: initialized, awaiting payment, submitted, verifying, confirmed, failed
- Transaction hash input with validation state
- Make the crypto flow feel trustworthy and easy, even for non-technical premium users

7. User dashboard
- Overview cards for bookings, purchases, recent activity, payment status
- Sections for profile, portfolio, orders, transactions
- Premium analytics visuals where relevant
- Strong hierarchy and clean navigation

8. Transaction history page
- Elegant table/list view
- Status badges
- Asset type, amount, network, date, hash preview, linked booking or asset
- Filters and search

9. Order tracking / payment tracking page
- Timeline or status-based UI
- Booking or purchase reference
- Current payment verification state
- Reconciliation / pending status treatment
- Clear support contact CTA

10. Profile/account page
- Personal details
- Security settings
- Preferred payment assets
- Saved traveler information
- Account verification feel

11. Admin dashboard
- Higher-density but still polished UI
- KPIs for bookings, revenue, payment confirmations, pending transactions
- Recent bookings and payment tables
- Operational clarity without losing premium branding

12. Sign in / authentication pages
- Elegant and minimal
- Premium brand presence
- Strong trust and security cues

UX requirements:
- Prioritize clarity and conversion
- Make high-value transactions feel safe
- Use concise, premium microcopy
- Keep forms easy to scan and complete
- Ensure dashboards are practical, not just beautiful
- Make CTAs obvious without being loud
- Include realistic empty/loading/success/error states
- Create responsive layouts for desktop, tablet, and mobile
- Accessibility should be strong: contrast, hierarchy, readable text, large tap targets

Style guardrails:
- Do not make it look like a generic banking app
- Do not make it look like a generic airline booking website
- Do not rely on purple-on-white startup aesthetics
- Do not use cheesy crypto imagery like coins floating everywhere
- Keep the design premium, restrained, and intentional

Output expectations:
- Create a coherent end-to-end product design system
- Use consistent components across marketing and app screens
- Make the landing page visually striking
- Make the app screens clean, premium, and highly usable
- Prioritize realism, polish, spacing, and hierarchy
- The final result should feel like “luxury aviation x institutional-grade crypto fintech”

If helpful, use a design language inspired by:
- private aviation brands
- luxury automotive configurators
- premium wealth management dashboards
- modern editorial layouts
But keep the final output unique and product-focused.

Please generate:
- a moodboard direction
- a mini style guide
- the main reusable components
- high-fidelity desktop screens
- responsive mobile variants for the most important flows
